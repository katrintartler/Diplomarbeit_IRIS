\section{Wärmesensotik testen}
Für die Simulation der Temperatur des Kindes, wurden im Testroboter Keramikwiderstände eingebaut, welche als Wärmequellen
genutzt werden. Dazu wurde der Aufbau und die Verschaltung mittels Steckbrett getestet. Besonders zu beachten ist die hohe Leistung 
die für eine Wärmesimulation nötig ist. Dementsprechend ist ein Test auf dem Steckbrett schwierig, da diese meisten nicht für hohe 
Leistung ausgelegt sind. Weiters ist auch die Vesorgung ein Problem, weil die Leistung über eine hohe Stromstärke erbracht werden muss.
\begin{align}
P &= I^2 \cdot R \notag \\
P &= \frac{U^2}{R}
\end{align}
Die verwendeten Keramikwiderstände haben einen Widerstandswert von 2,2$\Omega$. Die Verschaltung erfolgt mit 2 in Serie 
geschalteten Strängen mit jeweils 3 parallel geschalteten Keramikwiderständen. Siehe Abbildung \ref{fig:VerschaltungKRs}, einem Ausschnitt 
des Schaltplans der Platine aus \ref{Testroboter Platinen}. Somit ergibt sich ein Gesamtwiderstand von ca. 1,5$\Omega$. Als Spannungswert werden 7,4V eingesetzt. Diese entsprechen dem Spannungswert eines 2 Zellen 
Lithium-Ionen Akkus. Somit ist ein Nachrüsten auf eine eigenständige Spannungsversorgung möglich siehe \ref{Spannungsversorgung Testroboter}. 
\begin{figure}[htbp]
\centering
\includegraphics[width=10cm]{images/VerschaltungKRs.png}
\caption{Verschaltung der Wärmekomponenten}
\label{fig:VerschaltungKRs}
\end{figure}
\noindent
\begin{align}
P &= \frac{7,4^2}{1,5} = 36,507 W \notag \\ 
I &= \sqrt{\frac{36,507}{1,5}} = 5,0004 A
\end{align}
Der benötigte Strombedarf liegt bei unserer Leistung von 36,507W also bei ca 5A. 
Dieser Bedarf wird von einem 20A Labornetzteil gedeckt. Aufgrund der Wahl dieser Spannungsversorgung wurde beschlossen die weiteren 
Komponenten des Testroboters ebenfalls mit dem Netzteil zu betreiben.

\subsubsection{Interface}
Für die Installation der Pakete Bleak und Dear PyGui wird ein virtual enviornment verwendet. Das virtual enviornment (venv)
ist eine generiert Umgebung die bei der Entwicklung von eigenständiger Software viele
Vorteile bringt. In unserem Fall ist es vor allem um ein sauberes System zu belassen. Die Pakete die 
wir für das Interface und die Kommunikation brauchen werden nicht global installiert, sondern sind nur 
in dem entsprechenden Projekt zu finden. In weiterer Folge wäre es möglich das GUI in eine Programmdatei 
unzuwandeln. Hier ist es auch von Vorteil mit einem venv zu arbeiten. 
\noindent
Das graphische Interface ist mithilfe der Pythonlibray Dear PyGui (DPG) erstellt worden.
Der Aufbau des Graphischen Interfaces ist primär mit Buttons ausgestattet die bei einem Klick, eine Callback Funktion ausführen.

\begin{lstlisting}[language=C++]
    dpg.add_button[label = "Babycrying", callback = ble.backend.crying]
\end{lstlisting}

\noindent
Damit DGP ein Fenster erzuegen kann, müssen 6 Funktionen unbedingt im Programmcode enthalten sein. 
\begin{itemize}
    \item create context
    \item create viewpoint
    \item setup dearpygui
    \item show viewpoint
    \item start dearpygui
    \item destroy context
\end{itemize}

\noindent 
Wird eine dieser Funktionen nicht korrekt aufgerufen, führt das zu Problemen, weil das Interface nicht 
richtig durchlaufen kann. Die Initialisierung, Darstellung und Interaktion des GUI könnten sonst 
nicht realisiert werden. \cite{dpg_docu}
